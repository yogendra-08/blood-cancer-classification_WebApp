# Supabase Database Setup Guide
# Blood Cancer Classification Web App

## Step 1: Create Supabase Project
1. Go to https://supabase.com
2. Click "Start your project" 
3. Sign up/login with GitHub
4. Click "New Project"
5. Select your organization
6. Choose a database password (save it securely)
7. Select a region closest to you
8. Click "Create new project"
9. Wait for project to be ready (2-3 minutes)

## Step 2: Get API Keys
1. In your Supabase dashboard, go to Settings → API
2. You will find:
   - Project URL (something like https://xxxxxxxx.supabase.co)
   - anon public key (starts with eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...)
   - service_role key (keep this secret)

## Step 3: Create Database Table
1. In Supabase dashboard, go to Table Editor
2. Click "Create a new table"
3. Table name: predictions
4. Enable Row Level Security (RLS)
5. Add columns:

| Name | Type | Default | Description |
|------|------|---------|-------------|
| id | int8 | auto increment | Primary key |
| name | text | - | Patient name |
| age | int4 | - | Patient age |
| gender | text | - | Patient gender |
| mobile | text | - | Mobile number |
| predicted_class | text | - | Prediction result |
| confidence | float8 | - | Confidence score |
| input_image_url | text | - | Original image URL |
| gradcam_image_url | text | - | GradCAM image URL |
| created_at | timestamptz | now() | Creation timestamp |

6. Click "Save"

## Step 4: SQL Query Alternative
If you prefer SQL, run this in Supabase SQL Editor:

```sql
CREATE TABLE predictions (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT,
  age INTEGER,
  gender TEXT,
  mobile TEXT,
  predicted_class TEXT,
  confidence DOUBLE PRECISION,
  input_image_url TEXT,
  gradcam_image_url TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable RLS
ALTER TABLE predictions ENABLE ROW LEVEL SECURITY;

-- Allow anonymous inserts
CREATE POLICY "Allow anonymous inserts" ON predictions
  FOR INSERT WITH CHECK (true);

-- Allow anonymous reads  
CREATE POLICY "Allow anonymous reads" ON predictions
  FOR SELECT USING (true);
```

## Step 5: Setup Storage (for images)
1. In Supabase dashboard, go to Storage
2. Click "Create a new bucket"
3. Bucket name: predictions
4. Make bucket public
5. Go to Settings → Policies for this bucket
6. Add policy to allow uploads:

```sql
CREATE POLICY "Allow image uploads" ON storage.objects
  FOR INSERT WITH CHECK (
    bucket_id = 'predictions' AND 
    auth.role() = 'anon'
  );

CREATE POLICY "Allow image downloads" ON storage.objects  
  FOR SELECT USING (
    bucket_id = 'predictions' AND 
    auth.role() = 'anon'
  );
```

## Step 6: Configure Backend
1. Open backend/requirements.txt
2. Add these dependencies:
   ```
   supabase==2.3.0
   reportlab==4.0.7
   ```

3. Install dependencies:
   ```bash
   pip install supabase reportlab
   ```

4. Open backend/app.py
5. Add these imports at the top:
   ```python
   from supabase import create_client, Client
   from reportlab.pdfgen import canvas
   from reportlab.lib.pagesizes import letter
   from reportlab.lib.utils import ImageReader
   import io
   import base64
   ```

6. Add Supabase client initialization (replace with your keys):
   ```python
   # Supabase configuration
   SUPABASE_URL = "https://your-project-id.supabase.co"
   SUPABASE_KEY = "your-anon-key-here"
   supabase: Client = create_client(SUPABASE_URL, SUPABASE_KEY)
   ```

## Step 7: Environment Variables (Recommended)
Create a .env file in backend/:
```
SUPABASE_URL=https://your-project-id.supabase.co
SUPABASE_ANON_KEY=your-anon-key-here
SUPABASE_SERVICE_KEY=your-service-key-here
```

Then load them in app.py:
```python
from dotenv import load_dotenv
import os

load_dotenv()

SUPABASE_URL = os.getenv("SUPABASE_URL")
SUPABASE_KEY = os.getenv("SUPABASE_ANON_KEY")
```

## Step 8: Test Connection
Add this test endpoint to verify Supabase connection:
```python
@app.route('/test-db', methods=['GET'])
def test_db():
    try:
        response = supabase.table('predictions').select('*').limit(1).execute()
        return jsonify({"status": "connected", "data": response.data})
    except Exception as e:
        return jsonify({"status": "error", "message": str(e)})
```

## Step 9: Final Setup
1. Restart your Flask server
2. Test the /test-db endpoint
3. Your app is now ready to save predictions to Supabase!

## Notes:
- Keep your service_role key secret - never expose it in frontend
- The anon key is safe for frontend use
- Images will be stored in Supabase Storage bucket
- Database automatically timestamps entries
- You can view data in Supabase Table Editor
